# Maintenance Guide

## How To Run The Project

The repository does not include `package.json` or npm scripts. Serve the project with a local static HTTP server from the repository root:

```bash
python3 -m http.server 8000
```

Then open:

```text
http://localhost:8000
```

Using an HTTP server is recommended because the app fetches `src/data/artworks.json`.

## How To Add A New Artwork

1. Add the image file under `src/assets/images/`.
2. Add a new record to `src/data/artworks.json`.
3. Include required fields: `id`, `title`, `artist`, `year`, `technique`, `description`, `image`, `position`, and `size`.
4. Add `rotation` when placing artwork on a side or front-facing wall.
5. Add `video` or `audio` only when the asset is ready.
6. Run `node scripts/validate-artworks.js`.
7. Open the museum and test placement, label readability, hover, modal content, and guided tour behavior.

## How To Update An Artwork Image

1. Replace or add the image under `src/assets/images/`.
2. Update the `image` field in `src/data/artworks.json`.
3. Validate the catalog.
4. Test the artwork in the gallery and modal.

## How To Update A Cloudinary Video URL

1. Use a Cloudinary delivery URL, not a collection or admin URL.
2. Prefer optimized transformations:

```text
q_auto,f_auto,w_1280
```

3. Update the artwork's `video` field.
4. Validate JSON syntax.
5. Open the artwork modal and test playback.

## How To Add Or Edit Modal Content

Modal content is generated by `src/js/modules/UI/ArtworkPanel.js` from artwork metadata. Update metadata first when possible. Edit modal templates only when changing the structure of image, audio, or video presentation.

## How To Update Guided Tour Points

The current tour path is generated by `src/js/modules/Tour/tourPath.js` from artwork positions. To influence a stop:

- Adjust artwork `position` or `rotation`.
- Add optional `viewDistance`.
- Add optional `cameraHeight`.

For fully curated camera paths, extend the tour path logic carefully and document any new fields.

## How To Optimize Assets Before Adding Them

- Use compressed JPEG or WebP for artwork images where appropriate.
- Keep source image dimensions reasonable for browser texture memory.
- Keep videos in Cloudinary or another CDN-backed media service.
- Use Cloudinary transformations for quality, format, and width.
- Avoid committing large video files to Git.

## How To Test Before Deployment

Run:

```bash
node scripts/smoke-test.js
node scripts/validate-artworks.js
```

Also perform manual tests:

- Load the museum through a local HTTP server.
- Test free exploration.
- Test guided tour start, stop, and completion.
- Test artwork hover and selection.
- Test image, audio, and video modals.
- Test credits modal.
- Test desktop and mobile layouts.
- Check the browser console for errors.

## Common Mistakes To Avoid

- Opening `index.html` directly from the file system and expecting `fetch()` to work in every browser.
- Using a Cloudinary collection URL instead of a delivery URL.
- Adding video files directly to the repository.
- Forgetting to validate `artworks.json`.
- Placing artwork without matching wall rotation.
- Changing CSS selectors that JavaScript depends on.
- Adding new shadow-casting lights without measuring performance.
